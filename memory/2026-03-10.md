## ActivityWatch 连接器调试与修复 (11:43-12:08)
- **问题**: `aw-connect devices` 返回 HTTP 502 Bad Gateway
- **根本原因**: 系统代理设置（127.0.0.1:7890）导致 Python requests/urllib 无法直接访问 localhost:5600
- **诊断过程**:
  1. ActivityWatch 服务正常运行（进程存在，端口开放）
  2. curl 可以正常访问 API（返回 200）
  3. Python urllib 和 requests 都返回 502
  4. Debug 日志显示请求被路由到 127.0.0.1:7890（代理）
- **解决方案**:
  1. 修改 `rest_client.py` 使用 requests 库（更可靠）
  2. 为所有 localhost 请求禁用代理：`proxies={'http': None, 'https': None}`
  3. 修复 API 路径（ActivityWatch 需要尾部斜杠 `/buckets/`）
- **修复文件**: `/Users/mac/.openclaw/skills/aw-openclaw-connect/aw_client/rest_client.py`
- **测试结果**: ✅ 成功
  - `aw-connect devices` 返回设备列表
  - `aw-connect query` 可以查询活动数据
- **发现的设备**: macdeMac-mini.local (window watcher)
- **技术细节**:
  - Python 3.14 的 urllib 在处理某些 Flask 服务器时有兼容性问题
  - requests 库更稳定，但需要显式禁用系统代理
  - ActivityWatch API 对 URL 格式敏感（需要尾部斜杠）
