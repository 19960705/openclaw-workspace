## 系统概述

### 设计目标

构建一个自动化的内容生产系统，实现：
- **知识持久化**：所有灵感、素材、选题、成品永久保存在本地
- **流程标准化**：从灵感捕获到内容发布的完整 SOP
- **多平台适配**：支持 X/Twitter、小红书、公众号、TikTok 等
- **跨境电商支持**：泰区选品、趋势监控、产品分析
- **AI 工具分享**：工具评测、教程制作、案例分析

### 核心组件

- **Obsidian**：本地知识库，双向链接，Markdown 格式
- **OpenClaw**：7x24 后台运行，多 Agent 协作
- **obsidian-cli**：安全的文件操作工具
- **软链接**：多 Agent 共享同一知识库

## 核心架构

### 物理存储位置

```
/Users/mac/Documents/Obsidian Vault/Keonho/ContentFactory/
```

这是真实的物理存储位置，所有数据都保存在这里。

### 软链接映射

为每个 Agent 创建软链接：

```bash
# 主 Agent (Keonho)
ln -s "/Users/mac/Documents/Obsidian Vault/Keonho/ContentFactory" \
      /Users/mac/.openclaw/workspace/ContentFactory

# 代码 Agent
ln -s "/Users/mac/Documents/Obsidian Vault/Keonho/ContentFactory" \
      /Users/mac/.openclaw/agents/code-agent/ContentFactory

# 研究员 Agent
ln -s "/Users/mac/Documents/Obsidian Vault/Keonho/ContentFactory" \
      /Users/mac/.openclaw/agents/researcher-agent/ContentFactory

# 归档员 Agent
ln -s "/Users/mac/Documents/Obsidian Vault/Keonho/ContentFactory" \
      /Users/mac/.openclaw/agents/archivist-agent/ContentFactory
```

## 元数据规范

### 标准 YAML 头部

每个 Markdown 文件顶部必须包含 YAML 元数据：

```yaml
---
标题: 
阶段: 收件箱/灵感/选题/大纲/初稿/终稿/已发布
类型: AI工具/跨境电商/内容创作/个人成长
平台: X/小红书/公众号/TikTok/多平台
关联参考: [[相关笔记1]], [[相关笔记2]]
标签: #AI #跨境电商 #选品 #工具评测
创建日期: 2026-03-07
更新日期: 2026-03-07
状态: 待处理/进行中/已完成
优先级: 高/中/低
---
```

## Agent 配置

### 1. SOUL.md - 系统底层运行法则

```markdown
# SOUL.md - 系统底层运行法则

## 1. 绝对的操作护栏

- **严禁使用 Linux 原生的 mv 或 rm 命令操作 ContentFactory 目录下的任何文件**
- **所有涉及文件的移动、重命名、删除，必须且只能调用 obsidian-cli**
- 这是为了保护 Obsidian 的双向链接拓扑结构

## 2. 强制元数据注入

- 在 ContentFactory 目录下创建任何新的 .md 文件，文件最顶部必须包含标准 YAML 属性
- 禁止生成纯文本的无属性文件

## 3. 目录越权禁止

- 你的活动范围仅限于当前工作区和软链接的 ContentFactory
- 严禁尝试访问系统的其他目录
- 严禁修改点前缀（如 .git, .obsidian）的隐藏文件夹

## 4. 文件命名规范

- 文件名使用中文或英文，避免特殊字符
- 日期格式：YYYY-MM-DD
- 示例：`2026-03-07-OpenClaw内容工厂设计.md`

## 5. 安全确认机制

- 删除文件前必须确认
- 移动文件到已发布归档前必须确认
- 批量操作前必须列出清单供用户确认
```
