## 系统概述

### 设计目标

构建一个自动化的内容生产系统，实现：
- **知识持久化**：所有灵感、素材、选题、成品永久保存在本地
- **流程标准化**：从灵感捕获到内容发布的完整 SOP
- **多平台适配**：支持 X/Twitter、小红书、公众号、TikTok 等
- **跨境电商支持**：泰区选品、趋势监控、产品分析
- **AI 工具分享**：工具评测、教程制作、案例分析

### 核心组件

- **Obsidian**：本地知识库，双向链接，Markdown 格式
- **OpenClaw**：7x24 后台运行，多 Agent 协作
- **obsidian-cli**：安全的文件操作工具
- **软链接**：多 Agent 共享同一知识库

```markdown
# SOUL.md - 系统底层运行法则

## 1. 绝对的操作护栏

- **严禁使用 Linux 原生的 mv 或 rm 命令操作 ContentFactory 目录下的任何文件**
- **所有涉及文件的移动、重命名、删除，必须且只能调用 obsidian-cli**
- 这是为了保护 Obsidian 的双向链接拓扑结构

## 2. 强制元数据注入

- 在 ContentFactory 目录下创建任何新的 .md 文件，文件最顶部必须包含标准 YAML 属性
- 禁止生成纯文本的无属性文件

## 3. 目录越权禁止

- 你的活动范围仅限于当前工作区和软链接的 ContentFactory
- 严禁尝试访问系统的其他目录
- 严禁修改点前缀（如 .git, .obsidian）的隐藏文件夹

## 4. 文件命名规范

- 文件名使用中文或英文，避免特殊字符
- 日期格式：YYYY-MM-DD
- 示例：`2026-03-07-OpenClaw内容工厂设计.md`

## 5. 安全确认机制

- 删除文件前必须确认
- 移动文件到已发布归档前必须确认
- 批量操作前必须列出清单供用户确认
```

### 2. USER.md - 创作者偏好与风格设定

```markdown
# USER.md - 创作者偏好与风格设定

## 1. 个人信息

- **姓名**：Lunah (Jinny)
- **时区**：Asia/Shanghai
- **主要平台**：X/Twitter、小红书、公众号、TikTok
- **内容方向**：AI 工具、跨境电商、内容创作

## 2. 语言风格

### 拒绝的 AI 词汇
- 总而言之、不可否认、在这个瞬息万变的时代
- 赋能、降维打击、颠覆性、革命性
- 首先...其次...最后...、综上所述

### 喜欢的表达方式
- 短句，多用换行留白
- 真诚、专业、带一点极客的分享欲
- 用数据说话，用案例佐证
- 适当使用 emoji，但不过度

### 语气要求
- **X/Twitter**：简洁、犀利、有观点
- **小红书**：亲切、实用、接地气
- **公众号**：深度、专业、有逻辑
- **TikTok**：轻松、有趣、易懂

## 3. 排版习惯

- 喜欢用 > 引用框来突出金句
- 列表项一定要清晰，多用加粗 **强调核心关键词**
- 段落之间留白，避免大段文字
- 适当使用表格、图表

## 4. 内容偏好

### AI 工具类
- 重点：实用性、易用性、性价比
- 必须包含：使用场景、操作步骤、效果对比
- 避免：纯理论、过度吹捧

### 跨境电商类
- 重点：数据支撑、趋势分析、实操建议
- 必须包含：市场数据、产品案例、ROI 分析
- 避免：空谈、无数据支撑

### 内容创作类
- 重点：方法论、工具推荐、案例拆解
- 必须包含：具体步骤、可复制性、效果验证
- 避免：鸡汤、空洞理论

## 5. 质量标准

- **原创性**：必须有独特观点或新鲜角度
- **实用性**：读者看完能立即行动
- **数据性**：有数据、有案例、有对比
- **可读性**：结构清晰、逻辑严密、易于理解

### 3. SOP_CONTENT.md - 内容生产标准作业程序

```markdown
# SOP_CONTENT.md - 内容生产标准作业程序

当用户向你下达模糊指令时，请对照以下阶段执行相应的标准动作：

---

## 阶段 0：收件箱处理

**触发**：用户发送链接、文本、图片等原始素材

**动作**：
1. 将原始内容保存到 `00-收件箱/`
2. 自动提取关键信息（标题、来源、核心观点）
3. 打上临时标签
4. 询问用户：这是什么类型的素材？（AI工具/跨境电商/内容创作/其他）

**输出**：
- 文件：`00-收件箱/YYYY-MM-DD-临时笔记-序号.md`
- 元数据：阶段=收件箱，状态=待处理

---

## 阶段 1：灵感捕获与分类

**触发**：用户确认素材类型，或发送明确的灵感内容

**动作**：
1. 根据类型移动到对应的灵感库子目录：
   - AI 工具 → `01-灵感与素材库/3-AI工具库/`
   - 跨境电商 → `01-灵感与素材库/4-跨境电商素材/`
   - 其他 → `01-灵感与素材库/1-日常灵感剪报/`
2. 完善 YAML 元数据（类型、标签、关联参考）
3. 从中提炼 1-3 个可能的选题
4. 将选题保存到 `02-选题池/待写选题库/`
5. 在选题的 YAML 中链接原始灵感

**输出**：
- 灵感文件：完整的 YAML + 内容
- 选题文件：1-3 个新选题，阶段=选题

---

## 阶段 2：爆款素材沉淀

**触发**：用户发送金句、框架、案例，并标注"这是爆款素材"

**动作**：
1. 保存到 `01-灵感与素材库/2-爆款素材片段/`
2. 打上详细标签（如：#金句 #框架 #案例 #数据）
3. 标注适用场景（如：开头、结尾、论证、转折）
4. 建立与相关选题的双向链接

**输出**：
- 文件：`01-灵感与素材库/2-爆款素材片段/YYYY-MM-DD-素材名称.md`
- 元数据：阶段=灵感，类型=爆款素材

---

## 阶段 3：选题立项与大纲生成

**触发**：
- 用户问"今天写什么"
- 用户说"开始写 [选题名称]"
- 用户说"给我推荐选题"

**动作**：
1. 列出 `02-选题池/待写选题库/` 中的所有选题
2. 按优先级、类型、平台分组展示
3. 用户选定后，**必须先检索灵感库和爆款素材库**
4. 结合参考资料，生成 3 个不同切入点的大纲
5. 保存到 `03-内容工厂/1-大纲挑选区/`
6. 将选题移动到 `02-选题池/进行中选题/`

**严禁**：不经检索直接生成大纲

**输出**：
- 3 个大纲文件，每个包含：
  - 标题（3 个备选）
  - 核心观点
  - 结构框架（开头/中间/结尾）
  - 关联素材链接

---

## 阶段 4：初稿打磨

**触发**：用户确认某一大纲

**动作**：
1. 读取 `USER.md` 中的写作风格
2. 主动检索 `01-灵感与素材库/2-爆款素材片段/` 寻找合适的金句、案例
3. 根据目标平台调整格式：
   - **X/Twitter**：280 字以内，简洁犀利
   - **小红书**：1000-1500 字，分段清晰，多用 emoji
   - **公众号**：2000-3000 字，深度分析，逻辑严密
   - **TikTok**：脚本形式，15-60 秒，口语化
4. 生成初稿，保存到 `03-内容工厂/2-初稿打磨区/`
5. 自动检查：
   - 是否包含 AI 词汇（拒绝列表）
   - 是否有数据支撑
   - 是否有案例佐证
   - 段落是否过长

**输出**：
- 初稿文件，阶段=初稿
- 自检报告（可选）

---

## 阶段 5：多平台适配

**触发**：用户说"改写成 [平台] 版本"

**动作**：
1. 读取初稿或终稿
2. 根据目标平台特性改写：
   - **X/Twitter**：提炼核心观点，1-3 条推文
   - **小红书**：增加 emoji，调整排版，加封面图建议
   - **公众号**：保持深度，优化小标题，加引用框
   - **TikTok**：转换为脚本，标注镜头、文案、BGM
3. 保存到 `03-内容工厂/4-多平台适配/[平台]/`

**输出**：
- 平台适配版本，阶段=终稿

---

## 阶段 6：终稿确认

**触发**：用户说"这是终稿"或"可以发布了"

**动作**：
1. 将文件从初稿区移动到 `03-内容工厂/3-终稿确认区/`
2. 更新 YAML：阶段=终稿，状态=已完成
3. 询问用户：是否需要多平台适配？

**输出**：
- 终稿文件，等待发布

---

## 阶段 7：发布与归档

**触发**：用户说"已发布"或提供发布链接

**动作**：
1. 使用 `obsidian-cli move` 将文件移动到 `04-已发布归档/[平台]/`
2. 更新 YAML：
   - 阶段=已发布
   - 发布日期=今天
   - 发布链接=用户提供的链接
3. 将选题从 `02-选题池/进行中选题/` 移动到 `02-选题池/已完成选题/`
4. 建立双向链接：选题 ↔ 成品

**输出**：
- 归档文件，完整记录发布信息

---

## 阶段 8：数据复盘（可选）

**触发**：用户提供数据（浏览量、点赞、转发等）

**动作**：
1. 在归档文件的 YAML 中添加数据字段：
   ```yaml
   浏览量: 10000
   点赞数: 500
   转发数: 100
   收藏数: 200
   ```
2. 分析数据表现，提取成功要素
3. 将成功要素沉淀到 `05-知识图谱/内容创作方法论/`

**输出**：
- 数据复盘笔记
- 方法论更新

**Keonho**：
