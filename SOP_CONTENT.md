# SOP_CONTENT.md - 内容生产标准作业程序

当用户向你下达模糊指令时，请对照以下阶段执行相应的标准动作：

---

## 阶段 0：收件箱处理

**触发**：用户发送链接、文本、图片等原始素材

**动作**：
1. 将原始内容保存到 `ContentFactory/00-收件箱/`
2. 自动提取关键信息（标题、来源、核心观点）
3. 打上临时标签
4. 询问用户：这是什么类型的素材？（AI工具/跨境电商/内容创作/其他）

**输出**：
- 文件：`00-收件箱/YYYY-MM-DD-临时笔记-序号.md`
- 元数据：阶段=收件箱，状态=待处理

---

## 阶段 1：灵感捕获与分类

**触发**：用户确认素材类型，或发送明确的灵感内容

**动作**：
1. 根据类型移动到对应的灵感库子目录：
   - AI 工具 → `01-灵感与素材库/3-AI工具库/`
   - 跨境电商 → `01-灵感与素材库/4-跨境电商素材/`
   - 其他 → `01-灵感与素材库/1-日常灵感剪报/`
2. 完善 YAML 元数据（类型、标签、关联参考）
3. 从中提炼 1-3 个可能的选题
4. 将选题保存到 `02-选题池/待写选题库/`
5. 在选题的 YAML 中链接原始灵感

**输出**：
- 灵感文件：完整的 YAML + 内容
- 选题文件：1-3 个新选题，阶段=选题

---

## 阶段 2：爆款素材沉淀

**触发**：用户发送金句、框架、案例，并标注"这是爆款素材"

**动作**：
1. 保存到 `01-灵感与素材库/2-爆款素材片段/`
2. 打上详细标签（如：#金句 #框架 #案例 #数据）
3. 标注适用场景（如：开头、结尾、论证、转折）
4. 建立与相关选题的双向链接

**输出**：
- 文件：`01-灵感与素材库/2-爆款素材片段/YYYY-MM-DD-素材名称.md`
- 元数据：阶段=灵感，类型=爆款素材

---

## 阶段 3：选题立项与大纲生成

**触发**：
- 用户问"今天写什么"
- 用户说"开始写 [选题名称]"
- 用户说"给我推荐选题"

**动作**：
1. 列出 `02-选题池/待写选题库/` 中的所有选题
2. 按优先级、类型、平台分组展示
3. 用户选定后，**必须先检索灵感库和爆款素材库**
4. 结合参考资料，生成 3 个不同切入点的大纲
5. 保存到 `03-内容工厂/1-大纲挑选区/`
6. 将选题移动到 `02-选题池/进行中选题/`

**严禁**：不经检索直接生成大纲

**输出**：
- 3 个大纲文件，每个包含：
  - 标题（3 个备选）
  - 核心观点
  - 结构框架（开头/中间/结尾）
  - 关联素材链接

---

## 阶段 4：初稿打磨

**触发**：用户确认某一大纲

**动作**：
1. 读取 `USER.md` 中的写作风格
2. 主动检索 `01-灵感与素材库/2-爆款素材片段/` 寻找合适的金句、案例
3. 根据目标平台调整格式：
   - **X/Twitter**：280 字以内，简洁犀利
   - **小红书**：1000-1500 字，分段清晰，多用 emoji
   - **公众号**：2000-3000 字，深度分析，逻辑严密
   - **TikTok**：脚本形式，15-60 秒，口语化
4. 生成初稿，保存到 `03-内容工厂/2-初稿打磨区/`
5. 自动检查：
   - 是否包含 AI 词汇（拒绝列表）
   - 是否有数据支撑
   - 是否有案例佐证
   - 段落是否过长

**输出**：
- 初稿文件，阶段=初稿
- 自检报告（可选）

---

## 阶段 5：多平台适配

**触发**：用户说"改写成 [平台] 版本"

**动作**：
1. 读取初稿或终稿
2. 根据目标平台特性改写：
   - **X/Twitter**：提炼核心观点，1-3 条推文
   - **小红书**：增加 emoji，调整排版，加封面图建议
   - **公众号**：保持深度，优化小标题，加引用框
   - **TikTok**：转换为脚本，标注镜头、文案、BGM
3. 保存到 `03-内容工厂/4-多平台适配/[平台]/`

**输出**：
- 平台适配版本，阶段=终稿

---

## 阶段 6：终稿确认

**触发**：用户说"这是终稿"或"可以发布了"

**动作**：
1. 将文件从初稿区移动到 `03-内容工厂/3-终稿确认区/`
2. 更新 YAML：阶段=终稿，状态=已完成
3. 询问用户：是否需要多平台适配？

**输出**：
- 终稿文件，等待发布

---

## 阶段 7：发布与归档

**触发**：用户说"已发布"或提供发布链接

**动作**：
1. 使用 obsidian-cli 将文件移动到 `04-已发布归档/[平台]/`
2. 更新 YAML：
   - 阶段=已发布
   - 发布日期=今天
   - 发布链接=用户提供的链接
3. 将选题从 `02-选题池/进行中选题/` 移动到 `02-选题池/已完成选题/`
4. 建立双向链接：选题 ↔ 成品

**输出**：
- 归档文件，完整记录发布信息

---

## 阶段 8：数据复盘（可选）

**触发**：用户提供数据（浏览量、点赞、转发等）

**动作**：
1. 在归档文件的 YAML 中添加数据字段：
   ```yaml
   浏览量: 10000
   点赞数: 500
   转发数: 100
   收藏数: 200
   ```
2. 分析数据表现，提取成功要素
3. 将成功要素沉淀到 `05-知识图谱/内容创作方法论/`

**输出**：
- 数据复盘笔记
- 方法论更新

---

## 文件操作说明

### 移动文件

使用 `mv` 命令，保持文件名不变：

```bash
mv "源文件完整路径" "目标目录路径/"
```

示例：
```bash
mv "/Users/mac/.openclaw/workspace/ContentFactory/00-收件箱/2026-03-08-测试.md" \
   "/Users/mac/.openclaw/workspace/ContentFactory/01-灵感与素材库/1-日常灵感剪报/"
```

### 重命名文件

谨慎使用，重命名后需要手动更新所有引用：

```bash
mv "原文件路径" "新文件路径"
```

### 删除文件

必须先确认没有其他文件链接到它：

```bash
rm "文件路径"
```

### 创建双向链接

在 Markdown 文件中使用 `[[文件名]]` 格式：

```markdown
参考资料：[[2026-03-08-AI工具评测]]
```

注意：只写文件名，不要包含路径和 .md 扩展名。

## Perplexity 深度研究

### 使用场景

**阶段 1：灵感捕获**
- 用户提供话题 → Perplexity 搜索最新资讯
- 自动保存到灵感库

**阶段 3：选题立项**
- 选题确定后 → Perplexity 深度研究
- 收集数据、案例、趋势
- 生成参考资料

**阶段 4：初稿打磨**
- 需要数据支撑 → Perplexity 搜索
- 需要案例佐证 → Perplexity 查找
- 需要最新信息 → Perplexity 发现页

### 访问方式

通过 OpenClaw 无头浏览器访问：
```
browser action=open targetUrl="https://perplexity.ai" profile="openclaw"
```

### 可用模型

- **Sonar**：快速搜索
- **Gemini 3 Flash / 3.1 Pro**：深度分析
- **GPT-5.2**：复杂推理
- **Claude Opus 4.6**：长文本理解
- **Grok 4.1**：实时信息
- **Kimi K2.5**：中文优化

### 特殊功能

1. **发现页**：AI 日报、新闻聚合
2. **金融页**：市场数据 + Polymarket
3. **空间模板**：
   - 专利搜索
   - KOL 分析
   - 论文研究

### 工作流示例

**AI 工具评测**
1. Perplexity 搜索工具最新动态
2. 保存到 `01-灵感与素材库/3-AI工具库/`
3. 生成选题
4. Perplexity 深度研究（功能、价格、对比）
5. 写入大纲和初稿

**跨境电商选品**
1. Perplexity 搜索泰区趋势
2. 保存到 `01-灵感与素材库/4-跨境电商素材/`
3. 生成选题
4. Perplexity 深度研究（市场数据、竞品分析）
5. 写入大纲和初稿
